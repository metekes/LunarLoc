# Lunar Lander-Aided Visual-Inertial Localization Dataset

This repository contains the dataset accompanying the paper:

> **LAL-VINS: Lander-Anchored Visual-Inertial Navigation for Lunar Rovers**

## Dataset Overview

The dataset is generated using NVIDIA Isaac Sim and is designed for evaluating visual-inertial localization methods for multiple lunar rovers operating in the vicinity of a lunar lander.

Each sequence contains:

- One static lunar lander.
- Two lunar rovers (`cheri1` and `cheri2`).
- Monocular camera images.
- IMU measurements.
- Ground-truth odometry.
- TF transformations.
- Inter-rover ranging measurements.

For every sequence, three ROS bags are provided containing camera images at different frame rates:

- **25 Hz**
- **10 Hz**
- **5 Hz**

The original simulation produces camera images at **50 Hz**. The provided ROS bags are generated by temporally subsampling the original image stream to obtain image frequencies of 25 Hz, 10 Hz, and 5 Hz. All remaining sensor measurements, including IMU, odometry, TF, and range measurements, are identical across the three bags.

The IMU measurements are generated from the ground-truth motion using the following measurement model:

```math
\mathbf{a}_{\mathrm{meas}}=\mathbf{a}_{\mathrm{gt}}+\mathbf{a}_{n}+\mathbf{a}_{\mathrm{bias}}
```

```math
\boldsymbol{\omega}_{\mathrm{meas}}=\boldsymbol{\omega}_{\mathrm{gt}}+\boldsymbol{\omega}_{n}+\boldsymbol{\omega}_{\mathrm{bias}}
```

where the measurement noise is modeled as white Gaussian noise and the biases evolve as random walks.

The stochastic parameters used to generate the IMU measurements are:

| Parameter | Value |
|-----------|------:|
| Accelerometer noise standard deviation | `2 × 10⁻³ m/s²` |
| Gyroscope noise standard deviation | `7 × 10⁻⁴ rad/s` |
| Accelerometer bias random walk | `1 × 10⁻⁵ m/s³` |
| Gyroscope bias random walk | `2 × 10⁻⁶ rad/s²` |

## ROS Topics

Each ROS bag contains the following topics.

| Topic | Message Type | Description |
|-------|--------------|-------------|
| `/cheri1/camera/info` | `sensor_msgs/CameraInfo` | Camera calibration information for rover 1 |
| `/cheri1/camera/rgb/compressed` | `sensor_msgs/CompressedImage` | Monocular RGB images for rover 1 |
| `/cheri1/imu` | `sensor_msgs/Imu` | Simulated IMU measurements for rover 1 |
| `/cheri1/odom` | `nav_msgs/Odometry` | Ground-truth rover pose in the odometry frame |
| `/cheri1/tf` | `tf2_msgs/TFMessage` | TF tree for rover 1 |
| `/cheri2/camera/info` | `sensor_msgs/CameraInfo` | Camera calibration information for rover 2 |
| `/cheri2/camera/rgb/compressed` | `sensor_msgs/CompressedImage` | Monocular RGB images for rover 2 |
| `/cheri2/imu` | `sensor_msgs/Imu` | Simulated IMU measurements for rover 2 |
| `/cheri2/odom` | `nav_msgs/Odometry` | Ground-truth rover pose in the odometry frame |
| `/cheri2/tf` | `tf2_msgs/TFMessage` | TF tree for rover 2 |
| `/rover_distance` | `std_msgs/Float64MultiArray` | Inter-rover ranging measurements |

---

## Range Measurements

The topic

```text
/rover_distance
```

contains a `std_msgs/Float64MultiArray` with three elements:

| Index | Description |
|------:|-------------|
| 0 | Timestamp |
| 1 | Simulated range measurement with Gaussian noise (`σ = 0.05 m`) |
| 2 | Ground-truth distance between the two rovers |

The noisy range measurements can be directly used for evaluating localization methods employing inter-rover ranging sensors.

## Coordinate Frames

The TF tree contains the transformations required to recover the poses of the lander, rover, and camera.

### Camera Extrinsics

The camera extrinsic calibration is given by the transform

```text
frame_id: "base_link"
child_frame_id: "Camera_frame"
```

This transform defines the camera pose with respect to the rover body frame.

---

### Rover Pose

The rover ground-truth pose is given by

```text
frame_id: "odom"
child_frame_id: "base_link"
```

This transform represents the rover pose in its local odometry frame.

---

### Lander Pose

The lander pose in the world frame is given by

```text
frame_id: "world"
child_frame_id: "lander"
```

Since the lander is static, this transform remains constant throughout the sequence.

---

### Odometry Frame

The odometry frame with respect to the world frame is given by

```text
frame_id: "world"
child_frame_id: "odom"
```

Combining this transform with the rover pose yields the rover pose in the world frame.
